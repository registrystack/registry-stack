Create evidence verification

Verifies caller-submitted claims against the registry binding selected by the evidence offering. The offering determines the dataset, entity, lookup keys, and ruleset; callers cannot select legacy verification rulesets directly.

Authorizations:
bearerAuthapiKeyAuth
path Parameters
offering_id
required
string

Evidence offering identifier

header Parameters
Data-Purpose
string <uri> non-empty
Example: https://demo.example.gov/purpose/demo-review

Absolute purpose-of-use IRI recorded in the audit trail. Required by this entity's policy. Header names are case-insensitive, so Data-Purpose and data-purpose are equivalent. Required when the selected evidence offering's bound entity requires a purpose-of-use header.

Accept
string
Enum: "application/json" "application/vnd.registry-relay.evidence-verification+jwt"

Use application/json or omit for the default JSON response. Use application/vnd.registry-relay.evidence-verification+jwt to request a signed server-to-server receipt when provenance is enabled and the media type is configured.

Request Body schema: application/json
required
required
object

Caller-submitted facts. Per-ruleset shape is configured by the deployment; V1 matching is normalized_exact only.

Array of objects

Optional metadata about external documents or artifacts the caller already holds. Raw evidence is not echoed by default.

object

Optional targeted registry subject. The offering binding must explicitly allow targeted verification.

Responses

Configured base URL for this gateway instance.

http://civil-registry-relay:8080/evidence-offerings/{offering_id}/verifications

Civil Person

Operations on the civil_person entity in dataset civil_registry. Person-level civil status projection for evidence computation.

List records

List civil_person records from dataset civil_registry. Person-level civil status projection for evidence computation.

Supports pagination via limit+cursor, projection via fields, relationship expansion via expand, and configured filters.

Authorizations:
bearerAuthapiKeyAuth
query Parameters
limit
integer <int32> >= 1
Examples:
  • limit=10 -

Maximum records to return. Capped by the entity's api.max_limit.

cursor
string

Opaque pagination cursor returned in a prior response's pagination.next_cursor.

fields
string

Comma-separated list of entity fields to project. Unknown fields are rejected.

national_id
string

Filter by exact match on national_id.

national_id.in
string

Filter by inclusion in a comma-separated list of national_id values.

birth_date
string

Filter by exact match on birth_date.

birth_date.gte
string

Filter where birth_date is greater than or equal to the value.

birth_date.lte
string

Filter where birth_date is less than or equal to the value.

civil_status
string

Filter by exact match on civil_status.

civil_status.in
string

Filter by inclusion in a comma-separated list of civil_status values.

deceased
string

Filter by exact match on deceased.

district
string

Filter by exact match on district.

district.in
string

Filter by inclusion in a comma-separated list of district values.

header Parameters
Data-Purpose
required
string <uri> non-empty
Example: https://demo.example.gov/purpose/demo-review

Absolute purpose-of-use IRI recorded in the audit trail. Required by this entity's policy. Header names are case-insensitive, so Data-Purpose and data-purpose are equivalent.

Responses

Configured base URL for this gateway instance.
