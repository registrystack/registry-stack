This site is the documentation hub for the registry stack: the tutorials, how-to guides, product
docs, explanation, and reference for building with Registry Relay, Registry Notary, Registry
Manifest, Registry Atlas, Registry Platform, and Registry Lab.

It is organized as a Diataxis hub. Product detail is owned by each repository and pulled in at
build time, so the product pages always match the code. For the pitch (why registry surfaces
matter, who benefits, where the stack fits), see the marketing site at
[registrystack.org](https://registrystack.org/).

## Where to go

- New here? [Choose where to start](start/quickstart/) routes you to the right page, and
  [When to use Registry Stack](start/when-to-use/) covers what it is and when it fits.
- Want to try it locally? Start with
  [Publish a spreadsheet as a secured registry API](tutorials/publish-spreadsheet-secured-registry-api/),
  then [Verify a claim from your registry API](tutorials/verify-claim-registry-api/).
- Already have an API? Use
  [Verify a claim from your own API](tutorials/verify-claim-own-api/) to create a standalone Notary
  project.
- Want the full demo topology? Make [Your first call](start/your-first-call/), then take the full
  tour with [First run with Registry Lab](tutorials/first-run-with-registry-lab/).
- Working with one product? Open its docs under Products: [Registry Relay](products/registry-relay/),
  [Registry Notary](products/registry-notary/), [Registry Manifest](products/registry-manifest/),
  [Registry Atlas](products/registry-atlas/), [Registry Platform](products/registry-platform/), or
  [Registry Lab](products/registry-lab/).
- Need the model? Read the [architecture overview](explanation/architecture/) and
  [integration patterns](explanation/integration-patterns/).
- Need the registers? See the [API reference](reference/apis/), [standards register](reference/standards/),
  [contracts](reference/contracts/), [glossary](reference/glossary/), and
  [boundaries and map](map/boundaries-and-map/).

## Boundary

The pitch lives on [registrystack.org](https://registrystack.org/). This site keeps one concise
[when-to-use and non-goals](start/when-to-use/) page; everything else here is a tutorial, how-to,
product doc, explanation, or reference.
