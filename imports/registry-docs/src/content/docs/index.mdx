The registry family lets you do three things: describe a registry's published surface in every
standards format that downstream systems expect (Publish), query protected registry data over
authorized HTTP routes without gaining database access (Consult), and evaluate eligibility
claims and receive a verifiable credential without exposing the records that back the claim
(Prove).

## Where to start

- [Three stories](./start/what-we-built/): one page that maps each story to the projects that
  deliver it, with definitions.
- [Quickstart](./start/quickstart/): audience routing, pick the path that matches your goal.
- [Your first call](./start/your-first-call/): a reproducible tutorial using the Registry Lab
  demo, from `git clone` to a live credential in four steps.
