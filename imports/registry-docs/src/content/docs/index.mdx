This site is the documentation hub for the registry stack: the tutorials, how-to guides, product
docs, explanation, and reference for building with Registry Relay (protected read APIs over
existing registry data) and Registry Notary (privacy-preserving evidence and credentials).

It is organized as a Diataxis hub. Product detail is owned by each repository and pulled in at
build time, so the product pages always match the code. For the pitch (why registry surfaces
matter, who benefits, where the stack fits), see the marketing site at
[registrystack.org](https://registrystack.org/).

## Where to go

Work through three rungs, from zero install to going deep. Not sure which rung you want?
[Choose where to start](start/quickstart/) routes you to the right page, and
[When to use Registry Stack](start/when-to-use/) covers what it is and when it fits.

### Rung 1: See it live

[See it live](start/see-it-live/) is the fastest way to watch a protected registry read and a
signed credential, on a hosted lab with zero install.

### Rung 2: Run your own

Run a local single-node stack with the tutorials. Start with
[Publish a spreadsheet as a secured registry API](tutorials/publish-spreadsheet-secured-registry-api/),
then [Verify a claim from your registry API](tutorials/verify-claim-registry-api/). Already have an
API? Use [Verify a claim from your own API](tutorials/verify-claim-own-api/) to create a standalone
Notary project.

### Rung 3: Go deeper

Read the product docs for the two documented products:
[Registry Relay](products/registry-relay/) for protected read APIs over existing registry data, and
[Registry Notary](products/registry-notary/) for privacy-preserving evidence and credentials.
Behind them, the [architecture overview](explanation/architecture/) explains the model, and the
[API reference](reference/apis/) and [glossary](reference/glossary/) cover the details. For the
broader stack, see [registrystack.org](https://registrystack.org/).

## Boundary

The pitch lives on [registrystack.org](https://registrystack.org/). This site keeps one concise
[when-to-use and non-goals](start/when-to-use/) page; everything else here is a tutorial, how-to,
product doc, explanation, or reference.
