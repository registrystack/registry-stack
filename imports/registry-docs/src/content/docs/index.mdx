These docs cover the registry stack: six projects that let public authorities exchange
registry information across institutional boundaries while keeping each authority in
control of its own data.

The stack is organized around [three commitments and three capabilities](./start/what-we-built/):
interoperability, safeguards, and reviewability, delivered through Publish, Consult, and Prove.

## Pick your path

- **New here?** Read [Commitments and capabilities](./start/what-we-built/) for the mental model, then
  [Your first call](./start/your-first-call/) to run a real query against the demo.
- **Integrating with a registry?** Start at [Quickstart, "I want to integrate"](./start/quickstart/#i-want-to-integrate-with-a-registry).
- **Evaluating standards conformance?** Start at [Standards register](./reference/standards/).
- **Contributing to a project repo?** Start at the [Boundaries and map](./map/boundaries-and-map/)
  page to find the project that owns the question.
- **Operating a deployment?** Start at [Quickstart, "I'm operating"](./start/quickstart/#im-operating-a-deployment).
