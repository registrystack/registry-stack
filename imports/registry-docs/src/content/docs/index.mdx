This site documents the registry family: six projects, each with a clear job, that together let you publish, consult, verify, and operate protected registry data. If you're new to the family, start with [what is the registry family](./start/what-is-the-registry-family/).

## Where to start

Pick the entry point that matches what you want to do:

- **Want to see it run?** Start with the [Registry Lab tutorial](./tutorials/first-run-with-registry-lab/). One `docker compose up`, end-to-end demo.
- **Want to query a registry over HTTP?** Start with [Registry Relay](./projects/registry-relay/). It's the protected consultation gateway.
- **Need to evaluate eligibility claims and issue credentials?** Start with [Registry Witness](./projects/registry-witness/).
- **Maintaining shared security or operational behavior?** Start with [Registry Platform](./projects/registry-platform/).
- **Reviewing what a registry publishes?** Start with [Registry Atlas](./projects/registry-atlas/) for browser inspection, or the [standards register](./reference/standards/) for the per-standard stance.
- **Modeling registry metadata before any runtime exists?** Start with [Registry Manifest](./projects/registry-manifest/).
