Use this page to orient a review, integration, or contribution quickly.
For the smallest runnable demo, open the [Registry Lab tutorial](../../tutorials/first-run-with-registry-lab/).

## Read first

1. Start with [what is the registry family](../what-is-the-registry-family/) to identify the five repositories and their responsibilities.
2. Use the [project map](../../map/) when you need to identify the owning repository for a question.
3. Check the [standards register](../../reference/standards/) before writing a standards claim.
4. Check the [contracts index](../../reference/contracts/) before integrating with an API, schema, or generated artifact.
5. Run the [Registry Lab tutorial](../../tutorials/first-run-with-registry-lab/) when you need the smallest local demo.

## When to stop here

Stop and open the owning repository when the question is about implementation details,
runtime configuration, or service-specific contribution workflow.

Stay in Registry Legend when the question is about cross-project ownership, public terminology,
standards positioning, or published machine contracts.
