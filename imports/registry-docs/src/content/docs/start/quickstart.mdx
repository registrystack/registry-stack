Use this page when you are new to Registry Stack and need the shortest route to a useful answer.
These docs are organized around tasks and the Diataxis types (tutorials, how-to, explanation,
reference), not around repository names.

## The short model

Registry Stack has three current product surfaces.
Registry Relay makes existing registry-like data available through protected, read-only APIs.
Registry Manifest describes the registry surface before an integration is written.
Registry Notary answers configured evidence questions without exposing a full source record.

Stay in these docs for the cross-project architecture, product boundaries, standards evidence, API
contracts, and the product docs pulled from each repository.
Go to the marketing site at [registrystack.org](https://registrystack.org/) for the pitch: why
registry surfaces matter, who benefits, the sell-form use cases, and ecosystem positioning.
