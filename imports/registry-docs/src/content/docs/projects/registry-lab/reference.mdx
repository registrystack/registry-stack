This reference is hand-maintained; the source links in each table are the authoritative check.

## Compose service inventory

Full Compose file: [`compose.yaml`](https://github.com/jeremi/registry-lab/blob/main/compose.yaml).

| Service | Image / build | Role | Host port |
| --- | --- | --- | --- |
| `civil-registry-relay` | `registry-relay:demo` (local build) | Registry Relay for civil domain; serves CSV-backed civil registry data | 4311 |
| `social-protection-registry-relay` | `registry-relay:demo` (local build) | Registry Relay for social protection domain; serves XLSX-backed data | 4312 |
| `health-registry-relay` | `registry-relay:demo` (local build) | Registry Relay for health domain; serves Parquet-backed data | 4313 |
| `civil-witness` | `registry-witness:demo` (local build) | Registry Witness for civil claims; reads civil facts from `civil-registry-relay` over HTTP | 4321 |
| `social-protection-witness` | `registry-witness:demo` (local build) | Registry Witness for social protection claims; reads social facts from `social-protection-registry-relay` over HTTP | 4322 |
| `shared-eligibility-witness` | `registry-witness:demo` (local build) | Registry Witness for cross-authority claims spanning civil, social, and health registries | 4323 |
| `static-metadata-publisher` | `python:3.12.3-slim-bookworm` (pulled) | Python HTTP server serving the portable metadata static bundle at `/srv/static` | 4331 |
| `demo-client` | `python:3.12.3-slim-bookworm` (pulled) | Narrated demo client (one-shot, `--profile client` only); runs `scripts/demo-flow.py` | None |

Relay and Witness build contexts include `vendor/registry-platform` plus the owning service source
submodule, `vendor/registry-relay` or `vendor/registry-witness`.
All Relay and Witness services use `env_file: .env`, `RUST_LOG: info`, and `restart: unless-stopped`.
Relay services have a healthcheck (`registry-relay --help`, 30 s interval, 10 s timeout, 3 retries).
Witness services have no healthcheck in the compose file.
Inside the compose network, services reach each other via DNS names on port 8080.

Named cache volumes: `civil-registry-cache`, `social-protection-registry-cache`, `health-registry-cache`.

## Fixture file layout

Source: [`scripts/generate-fixtures.py`](https://github.com/jeremi/registry-lab/blob/main/scripts/generate-fixtures.py).
Run with `uv run scripts/generate-fixtures.py` from the repo root.
Output lands under `data/`. The directory is gitignored; only `data/.gitignore` is committed.

| File | Format | Domain | Contents |
| --- | --- | --- | --- |
| `data/civil/civil_registry.csv` | CSV | Civil | Children, caregivers, living adults, and deceased adults across five districts |
| `data/social-protection/social_protection_registry.xlsx` | XLSX | Social protection | Households, household members, and enrollments with active, inactive, suspended, and review-required cases |
| `data/health/health_registry.parquet` | Parquet | Health | Active, suspended, pending-renewal, and partially-serviceable facilities |

All timestamps are deterministic (2026-01-01).
The generator validates that a successful subject, failed predicates, deceased-member cases,
cross-source subjects, and health-linked support cases are all present before writing.

## Smoke-check scripts

Run `scripts/smoke.sh` after `docker compose -f compose.yaml up -d` to confirm the topology is healthy.
All smoke requests use the correlation ID `decentralized-demo-correlation-001`
(overridable via `DEMO_CORRELATION_ID`).

### smoke.sh

Source: [`scripts/smoke.sh`](https://github.com/jeremi/registry-lab/blob/main/scripts/smoke.sh)

More than 30 checks:

- Health and ready endpoints on all Relay services
- Discovery endpoints on all Witness services with bearer-token and `x-api-key` credentials
- OpenAPI endpoints on all services
- Evidence-offering endpoints
- Authorization scope denial (403 `auth.scope_denied`)
- Positive row-read and aggregate-read on social Relay
- Claim evaluation on civil, social, and shared Witnesses
- Cross-authority `source_count >= 2`
- Credential-bound SD-JWT VC evaluation
- Demo-client artifact presence
- Audit event log entries
- Absence of generated raw secrets in service logs

### release-check.sh

Source: [`scripts/release-check.sh`](https://github.com/jeremi/registry-lab/blob/main/scripts/release-check.sh)

Full release wrapper that runs in sequence: fixture generation, secret generation, static metadata publication, compose build, compose up, smoke run, demo client run, and compose down with `-v`.

| Witness client bearer (social) | `SOCIAL_EVIDENCE_CLIENT_BEARER` | Demo client and smoke: discovery and evaluation on `social-protection-witness` |
| Witness client bearer (shared) | `SHARED_EVIDENCE_CLIENT_BEARER` | Demo client and smoke: discovery and evaluation on `shared-eligibility-witness` |
| Audit hash secret (Relay) | `REGISTRY_RELAY_AUDIT_HASH_SECRET` | Relay audit redaction and keyed hashes |
| Audit hash secret (Witness) | `REGISTRY_WITNESS_AUDIT_HASH_SECRET` | Witness audit redaction and keyed hashes |
| Issuer JWK (all Witnesses) | `REGISTRY_WITNESS_ISSUER_JWK` | SD-JWT VC signing key; all three Witness configs use `issuer_key_env: REGISTRY_WITNESS_ISSUER_JWK` ([`config/witness/civil-witness.yaml` L43](https://github.com/jeremi/registry-lab/blob/main/config/witness/civil-witness.yaml#L43)) |

:::note[How credentials flow]
Relay configs reference only `*_HASH` env vars (SHA-256 fingerprints of the raw tokens).
Witness auth configs reference `*_HASH` env vars for client credentials.
Witness upstream source connections still reference `token_env` names for outbound calls to Relay.
No raw token is committed to the repository.
:::

## Static metadata publisher endpoints

Source: [`scripts/publish-static-metadata.sh`](https://github.com/jeremi/registry-lab/blob/main/scripts/publish-static-metadata.sh),
manifest at [`config/static-metadata/metadata.yaml`](https://github.com/jeremi/registry-lab/blob/main/config/static-metadata/metadata.yaml),
served by `static-metadata-publisher` at `127.0.0.1:4331`.
No authentication is required for any of these endpoints.

| Path | Content | Format |
| --- | --- | --- |
| `/metadata/index.json` | Static publication index listing all produced artifacts | JSON |
| `/metadata/catalog.json` | DCAT (Data Catalog Vocabulary) catalog for the demo registry | JSON |
| `/metadata/evidence-offerings.json` | Evidence offering descriptors pointing to Witness services | JSON |
| `/metadata/policies.jsonld` | ODRL (Open Digital Rights Language) policy documents | JSON-LD |

The bundle is produced by `registry-manifest-cli publish` from the portable manifest.
It does not include any Relay runtime details (source paths, table identifiers, scopes, cache paths).
The static manifest identifier `decentralized-demo-static-publication` is an intentionally stable
value set in `config/static-metadata/metadata.yaml` and is not renamed when the repository moves
to `registry-lab`.
