Each project in the registry family has its own home in Registry Legend.
Use this index when you know which product you want; use the
[Project map](../map/) when you do not.

Every project page set follows the same shape: an overview (what it is and where
it fits), one or more how-to pages (the tasks you actually do with it), and a
reference page (the surfaces it exposes). Evidence links into the upstream repo
appear inline; gaps are marked with `TODO[evidence]` notes rather than asserted.

## The five projects

- [Registry Manifest](./registry-manifest/): describe a registry once in YAML and render every standards-facing metadata artifact other systems need.
- [Registry Relay](./registry-relay/): expose protected tabular sources as authorized, read-only entity APIs with audit logging.
- [Registry Witness](./registry-witness/): evaluate eligibility claims against registry data and issue SD-JWT VC credentials with selective disclosure.
- [Registry Atlas](./registry-atlas/): inspect what a registry publishes in your browser and check strict capability matches against the standards it claims to emit.
- [Registry Lab](./registry-lab/): run three Relay instances, three Witness services, a static metadata publisher, and a narrated demo client together in one Docker Compose topology.
