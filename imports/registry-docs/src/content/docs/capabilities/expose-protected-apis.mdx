Expose protected APIs is the runtime consultation capability.
The primary implementation is Registry Relay, a config-driven HTTP gateway that maps CSV, XLSX,
Parquet, or PostgreSQL sources to public entity routes while keeping table identifiers, raw SQL,
and source paths out of public contracts.

The old v1 term is Consult.
In v2 language, Consult maps to protected API exposure: caller authentication, route-level scopes,
field projection, declared filters, aggregates, caller-scoped metadata, and tamper-evident audit
records.
Registry Platform supplies shared auth, OIDC, HTTP security, outbound HTTP, crypto, SD-JWT, and
audit primitives.

Use this capability when a partner system needs lawful read access to registry facts without
direct database access, write paths, or unscoped bulk export.
Relay is not an open-data portal.
