The v2 capability names describe what the registry family lets an operator or reviewer do:
describe registries, expose protected APIs, certify evidence, inspect published artifacts, and
audit and operate the runtime surface.
These pages are lookup entries, not narrative entry pages.

The older terms still appear in v1 navigation and existing explanations.
Use this mapping when moving between v1 and v2 language.

| Old term | v2 capability | Primary project |
| --- | --- | --- |
| Publish | Describe registries | Registry Manifest |
| Consult | Expose protected APIs | Registry Relay |
| Prove | Certify evidence | Registry Witness |
| n/a | Inspect published artifacts | Registry Atlas |
| n/a | Audit and operate | Registry Relay, Registry Witness, Registry Platform |

## Capability pages

- [Describe registries](./describe-registries/): metadata, portable manifests, and generated
  discovery artifacts.
- [Expose protected APIs](./expose-protected-apis/): caller-scoped, read-only registry access
  through Registry Relay.
- [Certify evidence](./certify-evidence/): claim evaluation, disclosure control, and credential
  issuance through Registry Witness.
- [Inspect published artifacts](./inspect-published-artifacts/): Registry Atlas as a supporting
  inspection tool for generated metadata.
- [Audit and operate](./audit-and-operate/): audit envelopes, shared Platform primitives, and the
  future-facing Trust Control Plane boundary.
