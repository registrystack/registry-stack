Describe registries is the metadata capability.
The primary implementation is Registry Manifest, which compiles a portable `metadata.yaml`
manifest into catalogs, schemas, service descriptions, policy documents, evidence offerings,
and an `index.json` discovery entry point.

The old v1 term is Publish.
In v2 language, Publish maps to metadata description rather than deployment or data release.
Registry Manifest does not read registry records, open an HTTP listener, or grant credentials.
Registry Relay can also serve Manifest-rendered metadata at runtime, filtered to the authenticated
caller's metadata scopes.

Use this capability when a registry needs a machine-readable declaration of what exists, which
standards-shaped artifacts are emitted, which evidence offerings are advertised, and which runtime
service is expected to enforce access.
