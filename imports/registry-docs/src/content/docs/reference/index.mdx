Use these pages to look up specific facts: contract surfaces, standards claims, API routes, and shared terms. Each reference page links to the source code or data file that backs every claim.

If you're integrating with Registry Relay, start with the API reference; for standards conformance claims, the standards register; for cross-project contracts, the contracts page.

- [Contracts](./contracts/) lists the machine-readable surfaces each project exposes, their status, and where the source artifact lives.
- [Standards](./standards/) lists every standards claim across the registry family, with an explicit claim level and supporting evidence.
- [API references](./apis/) links to generated Redoc pages for [Registry Relay](../../projects/registry-relay/) and [Registry Witness](../../projects/registry-witness/).
- [Glossary](./glossary/) defines product names and domain terms used across v0 pages.
