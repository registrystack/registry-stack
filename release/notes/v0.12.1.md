# Registry Stack v0.12.1

Registry Stack v0.12.1 is the beta-15 fix-forward release after the v0.12.0
publication attempt stopped at the image advisory gate. The v0.12.0 tag remains
an immutable historical marker and did not produce a completed GitHub Release.
Beta-15 preserves the beta-14 product scope and adds only the confirmed release
blocker fix and release-process hardening.

## Fix-forward changes

- Registry Relay and Registry Notary now carry explicit reviews for the three
  non-fixable Debian 13 `libc6` findings discovered in the v0.12.0 candidate
  images. The policy binds each review to the ordered runtime root filesystem
  layer digest, rejects fixable, expired, mismatched, or unreviewed findings,
  and keeps the exact scanned image digest and source revision as evidence.
- Release preparation and finalization now have deterministic read-only plans
  for an explicit version and release ID. Planning rejects reused identities,
  verifies tag absence against the remote instead of trusting a potentially
  stale local tag set, and fails closed when remote tag state cannot be read.
- Release CI now uses conservative path routing, cancels superseded pull-request
  runs without cancelling release work, records binary-build timing, and uses
  a release-scoped Cargo cache.
- A real Buildx smoke builds both release Dockerfiles and checks the expected
  Open Containers Initiative source and revision labels before an immutable
  tag is created. This smoke is structural; the exact final candidate images
  are still scanned against their product-owned advisory policy before tagging.

## Product scope

There are no new product features relative to v0.12.0. Registry Notary's
wallet-facade, status-verification, bounded-batch, and generated-configuration
schema changes; Registry Relay's stable-surface roster, refresh-health, and
Debian 13 changes; and Registryctl's output and project-publication changes are
the unchanged beta-14 scope. Use the v0.12.0 notes for the full migration
detail, but install and deploy the completed v0.12.1 artifacts.

Registry Manifest has no user-visible change in this train. Crosswalk remains
pinned at `1d44ec735fdc8a7c719264b339574371e8330337`.

## Release evidence and held gates

The release workflow retains the eight-artifact public inventory: Registry
Notary, the Notary CEL worker, Registry Relay, the Relay Rhai worker, Registry
Manifest CLI, Registryctl, the Registryctl image lock, and Registry Docs. It
also publishes the three runtime images, checksums, Software Bill of Materials
files, vulnerability reports, a release capsule, signatures, and provenance.

The reviewed Debian findings are time-bounded accepted risk, not a claim that
the base operating system has no known vulnerabilities. A fixable finding, a
changed runtime root filesystem, an expired review, or a finding outside the
reviewed set fails the advisory gate.

Solmara Lab repinning, the complete upgrade and rollback exercise, OpenCRVS and
DHIS2 external runs, OpenID conformance, external wallet or verifier
interoperability, hosted publication, and the independent pilot remain held.
This source note does not claim any of those results.
