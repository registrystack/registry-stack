# Standards and Specification Claims Inventory

Issue: [#199](https://github.com/registrystack/registry-stack/issues/199)

This is the release-readiness inventory for standards and specification claims
in the docs site, product docs, and crate READMEs. It records the claim wording
as it exists today and classifies evidence as `none`, `test vectors`,
`conformance run`, or `interop`.

The evidence status is intentionally narrow. A `conformance run` row means a
selected artifact or profile has saved external validator evidence, not that the
whole product is certified. An `interop` row means a live external implementation
or product API is part of the documented proof. Rows marked `none` either have
no saved evidence yet or are explicit non-claims and comparison boundaries.

## Follow-Up Notes

- The generated standards register is broad but does not cover every crate and
  product README claim. This inventory adds those surfaces explicitly.
- OID4VCI Draft 13 should stay scoped until the implementation evidence anchors
  the draft revision as directly as the product docs do.
- WCAG, OpenSSF, SLSA, DPI Safeguards, GDPR, OOTS, eIDAS, GovStack, and X-Road
  rows are claim-boundary rows. They should prevent accidental upgrade to
  conformance language.
- `release/READINESS.md` should treat the inventory as complete, while leaving
  per-claim evidence review open until maintainers accept or downgrade the
  statuses above.
