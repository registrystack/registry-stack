# Code Of Conduct

Registry Stack is built for public review, institutional evaluation, and
responsible collaboration. Everyone participating in issues, pull requests,
discussions, reviews, or other project spaces is expected to help keep the
project useful, direct, and respectful.

## Expected Behavior

- Be clear, kind, and specific.
- Critique code, claims, and decisions without personal attacks.
- Assume public conversations may be read by adopters evaluating the project.
- Respect privacy boundaries around registry data, credentials, deployment
  details, and security reports.
- Accept maintainer decisions about scope, timing, and release readiness.

## Unacceptable Behavior

- Harassment, threats, insults, or sustained disruption.
- Publishing private information, credentials, registry records, or deployment
  details without explicit permission.
- Opening public reports for sensitive security issues covered by
  [SECURITY.md](SECURITY.md).
- Pressuring maintainers for support, review, or release commitments outside
  the project's stated scope and capacity.

## Enforcement

Maintainers may edit, hide, lock, or delete comments; close issues or pull
requests; block users; or report abuse to GitHub when behavior harms the
project or its users. Enforcement decisions are made to protect the work and the
people relying on it.
